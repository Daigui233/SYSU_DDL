# SYSU_DDL 仓库说明

仓库用途

本仓库用于中山大学 DDL_大电流 车队的上下位机协同开发与版本管理。

当前仓库按照上下位机进行划分，后续队员可围绕各自模块独立开发，再通过 Git 进行合并与审查。

目录结构

  • Master_RK3588S/
    上位机工程目录。当前将逐步接入基于 Linux / RK3588S 平台的通信、感知与决策代码。

  • Slave_TC264D/
    下位机工程目录。当前为主要开发对象，基于逐飞 TC264D 开源库进行二次开发。

当前协作约定

  • 下位机源码保持与逐飞库一致，使用 GBK 编码
  • 模块按功能逐个搭建、逐个审查
  • 每次修改尽量只围绕单一模块展开
  • 优先保证代码结构清晰、注释明确、职责边界稳定

当前开发重点

当前阶段主要围绕下位机开展工作，目标是：

  • 先完成椭圆环赛道稳定巡航
  • 同时为后续 OCR、避障、红绿灯、限速、金币等任务预留接口
  • 建立便于多人协作维护的控制链路和通信框架

当前方案审查

结合当前上下位机第一版代码，现阶段方案已经能够支撑“先打通链路、先准备椭圆巡线”的目标，但后续仍有若干必要或推荐改进项，建议在后续迭代中重点关注。

必要改进

  • 下位机 Control / Communication / State / PID 当前虽然均已通过 ADS 编译，但仍未完成人工审查与实车验证，后续必须结合实际联调继续收口。
  • 下位机控制周期当前采用主循环中的固定 10ms 软调度，而非 PIT 定时中断硬周期；当前可用，但后续任务增多后可能出现周期抖动，建议后续升级为定时中断驱动。
  • 下位机串口接收当前采用轮询固定帧解析，现阶段便于联调，但后续若通信频率、任务复杂度或协议长度继续增长，建议改为串口中断 + 缓冲区方案。
  • 下位机已加入 3s 掉线保护并自动切换到 STATE_SAFE_STOP，但该保护逻辑仍需后续通过实际断链测试验证是否满足安全预期。
  • 下位机状态切换当前主要信任上位机 state_cmd，本地下位机保护状态仲裁仍不充分；后续如加入急停、异常保护、任务接管，建议补本地状态优先级机制。
  • 上位机当前 main.c 仅为椭圆巡线最小主循环骨架，track_error 仍为占位接口；后续必须在模型、视觉或巡线程序接入后，明确误差定义并正式替换该入口。
  • 上位机第一版 C 通信代码尚未经过 RK3588S Linux 实机编译与串口联调验证，后续必须在板端实际跑通 make、串口打开、收发和断链场景。

推荐改进

  • 当前固定帧协议适合第一版联调，但扩展性一般；后续若 OCR、红绿灯、限速、避障、金币等字段显著增多，建议升级为更清晰、可扩展的协议格式。
  • 下位机 Control 模块中各状态参数当前直接写在 switch 内，现阶段直观但不利于长期维护；后续可考虑整理成独立参数表。
  • 下位机反馈帧当前已能回传速度、电机输出、舵机输出、状态值；后续可视调试需要补充圈数、错误码、掉线状态、任务状态等信息。
  • 上位机当前采用 C 完成通信层是合理的，但后续若视觉/模型主流程以 Python 为主，可再决定采用“Python 直接发串口”还是“Python 提供误差，C 负责串口发送”的最终架构。
  • 当前上下位机 README 已补充协议表、编译命令和运行说明；后续每次协议字段、状态语义或超时策略调整后，建议同步更新三份 README，避免代码与文档脱节。

当前通信协议

当前上下位机通信协议先采用固定帧方案，目的是先尽快打通 RK3588S <-> TC264D 的控制链路。
后续随着 OCR、红绿灯、限速、避障、金币等任务接入，协议字段和帧格式仍需要继续迭代。

串口参数

当前上下位机串口参数如下：

参数 当前值 备注
下位机串口模块 UART_1 逐飞库串口1
波特率 460800 当前固定值
下位机 TX P20.10 对应 UART1_TX_P20_10
下位机 RX P33.13 对应 UART1_RX_P33_13
供电 3.3V 与板上串口1接口一致
地线 GND 上下位机必须共地

实际接线时遵循串口交叉连接原则：

  • 上位机 TX 接 下位机 RX
  • 上位机 RX 接 下位机 TX
  • GND 共地

上位机 -> 下位机控制帧

  • 帧头：0x42
  • 地址：0x10
  • 固定总长度：14 字节
字节偏移 字段名 类型 长度 当前值/范围 说明
0 frame_head uint8 1 0x42 固定帧头
1 addr uint8 1 0x10 上位机发往下位机地址
2 payload_len uint8 1 10 当前数据区长度
3~6 track_error float 4 - 赛道误差/路径误差
7~10 target_speed float 4 - 目标速度
11 state_cmd uint8 1 状态枚举值 上位机下发状态
12 flags uint8 1 位标志 预留给急停、红灯、限速、避障等
13 checksum uint8 1 累加和低 8 位 对 0~12 字节做校验

下位机 -> 上位机反馈帧

  • 帧头：0x42
  • 地址：0x90
  • 固定总长度：15 字节
字节偏移 字段名 类型 长度 当前值/范围 说明
0 frame_head uint8 1 0x42 固定帧头
1 addr uint8 1 0x90 下位机发往上位机地址
2 payload_len uint8 1 11 当前数据区长度
3~6 actual_speed float 4 - 当前实际速度反馈
7~10 motor_output int32 4 - 当前电机输出控制量
11~12 servo_output uint16 2 - 当前舵机输出占空比
13 state_value uint8 1 状态枚举值 当前下位机状态
14 checksum uint8 1 累加和低 8 位 对 0~13 字节做校验

宏名 当前值 所在文件 含义

COMM_RX_PAYLOAD_LEN 10 Slave_TC264D/code/Communication.h 控制帧数据区长度
COMM_RX_FRAME_LEN 14 Slave_TC264D/code/Communication.h 控制帧总长度
COMM_TX_PAYLOAD_LEN 11 Slave_TC264D/code/Communication.h 反馈帧数据区长度
COMM_TX_FRAME_LEN 15 Slave_TC264D/code/Communication.h 反馈帧总长度

当前阶段说明

  • 当前协议是第一版，重点是先打通链路，不是最终版协议
  • 当前采用固定帧而非变长帧，优点是实现简单、联调快，缺点是扩展性一般
  • 后续如果状态字段、感知字段、调参字段明显增多，建议再升级为更清晰的可扩展协议格式
